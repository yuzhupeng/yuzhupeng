# Claude Code 开发Agent/工程项目专用提问技巧+全套提示词模板
## 一、核心提问底层技巧（先掌握，再套提示词）
### 1. 角色前置技巧
固定给Claude Code锁定**工程开发专属身份**，禁止通用闲聊，强制输出工程化、可落地代码/架构文档。
标准身份句式：
> 你现在是资深全栈AI Agent架构师，精通Claude Code、Python/TS/C#、工具调用、多智能体协作、工程落地、单元测试、部署脚本，只输出工业级可运行代码、规范架构、完整流程，拒绝伪代码、简略方案、模糊描述。

### 2. 约束三要素（每次提问必加，大幅提升输出质量）
1. **输出格式约束**：指定Markdown、分模块、代码带完整注释、分层目录结构；
2. **工程落地约束**：要求可直接复制运行、包含依赖清单、异常捕获、日志、配置文件；
3. **边界约束**：明确技术栈、运行环境、项目规模（Demo/生产级Agent）、禁止简化关键逻辑。

### 3. 分层提问策略（Agent项目专用，避免一次提问信息过载）
1. 第一层：架构设计（整体流程、多Agent分工、工具调用链路）
2. 第二层：核心模块编码（规划Agent、工具Agent、记忆模块、RAG模块）
3. 第三层：配套工程（配置、测试、日志、Docker、API服务）
4. 第四层：调优&排错（性能优化、上下文截断、API限流、报错修复）

### 4. 上下文复用技巧
每次追加需求时带上：上一轮项目名称、技术栈、已有代码片段，让Claude Code无缝衔接迭代，不用重复复述项目背景。

### 5. 避坑提问规则
- 禁止模糊提问：“帮我写个Agent” → 改为“基于Claude Code，开发带工具调用、短期记忆的单任务自动化Agent，技术栈Python 3.11”
- 强制要求错误处理：所有代码必须加try-except、参数校验、超时控制
- 强制分文件：复杂Agent项目要求输出完整项目目录，每个文件独立代码块

## 四、进阶高阶提问技巧（大幅提升Claude Code输出专业度）
### 1. 强制规范输出指令（加在任意提示词末尾）
```
附加强制规则，必须全部遵守：
1. 不使用省略号、不写# 省略逻辑，所有分支完整实现；
2. 配置项全部抽离至.env，禁止代码硬编码密钥、路径、模型参数；
3. 所有函数增加清晰入参、返回值类型注解；
4. 复杂逻辑拆分独立函数，单个函数代码不超过80行；
5. 输出全部中文注释，不使用英文简写无说明；
6. 最终交付内容按：架构说明 → 目录结构 → 代码 → 操作步骤 顺序输出。
```

### 2. 限制幻觉专用指令（解决Agent代码逻辑漏洞）
```
重要约束：禁止编造不存在的库、API、函数、工具；所有代码逻辑必须可验证，Claude不熟悉的工具需给出替代开源方案；若存在技术不确定性，明确标注风险点，不模糊带过。
```

### 3. 企业生产级强化指令（商用项目必加）
```
按企业生产标准开发：包含参数校验、权限控制、日志分级、异常告警、数据持久化备份、兼容多环境，不使用仅适用于本地Demo的简化写法。
```
